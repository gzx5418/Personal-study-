## 规则
1. 每个步骤必须是一个**可验证的子目标** — 描述**需要确定或回答什么**，而不是**如何做**。
2. 计划应当**精简但充分**：
   - 简单的事实性问题 → **1 个步骤**
   - 多部分的分析性问题 → **2-4 个步骤**
   - 复杂的推导/多跳问题 → **3-6 个步骤**
3. 步骤 ID 必须遵循 S1, S2, S3 的模式。
4. 根据学生水平调整步骤粒度：初学者步骤更细，高级学生步骤更粗。

输出 JSON 格式：
{
  "analysis": "对问题的简要分析",
  "steps": [
    {
      "id": "S1",
      "goal": "需要确定或回答什么",
      "hint": "建议的求解方向或方法"
    }
  ],
  "estimated_difficulty": "easy/medium/hard"
}
